\doxysubsection{Detailed Description}
REST API server for managing MSFS \doxylink{namespaceDIS}{DIS} Bridge sessions. 

The \doxylink{classRestServer}{Rest\+Server} class provides a web-\/based REST API interface for managing simulation sessions in the MSFS \doxylink{namespaceDIS}{DIS} Bridge system. It allows remote clients to start, stop, and monitor bridge sessions through HTTP requests.

The server provides endpoints for\+:
\begin{DoxyItemize}
\item Session management (start/stop operations)
\item Status monitoring and health checks
\item Configuration and diagnostic information
\item Client IP-\/based configuration mapping
\end{DoxyItemize}

The server uses the Crow web framework for HTTP handling and integrates with the \doxylink{classSessionManager}{Session\+Manager} for actual session operations. Client requests can be mapped to specific configurations based on their IP addresses.

\begin{DoxyNote}{Note}
This server is designed to run in a blocking mode and should typically be executed in its own thread when integrated into a larger application. 
\end{DoxyNote}

Start the REST server (blocking operation) 

Starts the HTTP server and begins listening for client requests. This method blocks the calling thread until the server is stopped (typically via signal or external shutdown mechanism).

The server will handle incoming HTTP requests and route them to appropriate handlers for session management operations.

\begin{DoxyNote}{Note}
This method blocks indefinitely. In multi-\/threaded applications, consider running this in a dedicated thread.
\end{DoxyNote}
\begin{DoxyWarning}{Warning}
The server will continue running until explicitly stopped or the process is terminated. Ensure proper shutdown mechanisms are in place. 
\end{DoxyWarning}
