Detailed Description

The Decode class allows the bridge to tranform incoming PDU packets back into the FlightData structure.

Warning
NOT CURRENTLY IMPLEMENTED - This class is for the future implementation of a two way bridge

The Decode class provides the functionality to tranform incoming DIS packets back into FlightData structures. This would transform the current one-way bridge into a two-way one.

It is the reverse operation of the Encode class.

Decode::Decode ( MappingConfig & config)

Constructs a Decode object with the specified mapping configuration.

Initializes the decoder with a reference to a MappingConfig object that defines how DIS PDU fields should be mapped back to FlightData structure fields. The configuration must remain valid for the lifetime of the Decode object.

FlightData Decode::decodePacket ( const std::vector< uint8_t > & buffer)

Decodes a DIS PDU into a FlightData structure.

Takes a byte buffer containing a PDU and converts it back into a FlightData struct

Parameters
bufferByte vector containing the PDU data received from the network. The buffer should contain a complete DIS PUD in network byte order.
Returns
FlightData structure with decoded aircraft state information.
Exceptions
std::runtime_errorif the PDU cannot be parsed from the buffer
std::runtime_errorif the mapping configuration fails to process the PDU
std::invalid_argumentif the buffer is empty or contains invalid data
Note
The input buffer is expected to be in network byte order (big-endian)
Only Entity State PDUs are currently supported for decoding
